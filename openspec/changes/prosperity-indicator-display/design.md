## Context

当前游戏引擎每季度调用 `WalkProsperity()` 更新各行业景气度，景气值存储在 `industry_prosperity` 表，售价、营收、估值等计算均依赖景气度。但原始景气值完全不暴露给前端，玩家只能通过销量浮动间接感知，无法理解变化原因。

## Goals / Non-Goals

**Goals:**
- 在经营指标面板展示景气度季度变化百分比
- 涨绿色、跌红色，与 UI 颜色约定一致
- 百分比保留 2 位小数，末尾省略 0（如 3.50% → 3.5%）

**Non-Goals:**
- 不新增 API 路由或 WebSocket 消息
- 不新增 DB 查询方法（复用已有 `LatestProsperity` / `ProsperityHistory`）
- 不涉及行业总览页或独立景气仪表盘
- 不修改游戏引擎逻辑

## Decisions

| 决策 | 选择 | 理由 |
|------|------|------|
| 数据源 | 复用 `GET /api/company/state` 现有响应 | 公司页已有该请求，不增加额外网络开销 |
| 上期值获取 | `ProsperityHistory(industry, 2)` 取第 2 条 | 已有函数，limit=2 只需查 2 行 |
| 变化率计算 | `(当期 - 上期) / 上期 × 100%` | 最直观的百分比变化表达 |
| 展示位置 | 经营指标网格第 6 项（库存之后） | 该面板已展示公司运营核心数据，景气度是需求端的关键解释变量 |
| 颜色 | 正 → `text-accent-green`，负 → `text-accent-red` | 沿用项目现有的中国红绿约定 |

## Risks / Trade-offs

- **上期值不存在时**（如公司刚成立、行业刚启用）：退化显示 `—`，不抛错
- **上期值为 0**：理论不可能（景气中心值为 1.0，边界 > 0），但做除零保护
