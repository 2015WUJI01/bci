## Context

当前游戏引擎每季度调用 `WalkProsperity()` 更新各行业景气度，景气值存储在 `industry_prosperity` 表，售价、营收、估值等计算均依赖景气度。但原始景气值完全不暴露给前端，玩家只能通过销量浮动间接感知，无法理解变化原因。

## Goals / Non-Goals

**Goals:**
- 在经营指标面板展示"需求量"：上季实际销量 + 景气度偏离百分比
- 涨红色、跌绿色，与 A 股股票颜色约定一致
- 百分比保留 2 位小数，末尾省略 0（如 23.40% → 23.4%）
- 上季结算不存在时回退为仅显示景气度偏离

**Non-Goals:**
- 不新增 API 路由或 WebSocket 消息
- 不新增 DB 查询方法（复用已有 `LatestProsperity` / `ProsperityHistory`）
- 不涉及行业总览页或独立景气仪表盘
- 不修改游戏引擎逻辑

## Decisions

| 决策 | 选择 | 理由 |
|------|------|------|
| 数据源 | 复用 `GET /api/company/state` 现有响应 | 公司页已有该请求，不额外增加网络开销 |
| 景气值含义 | 相对于基准 1.0 的偏离百分比 `(prosperity - 1) × 100%` | 景气本身就是百分比含义的数字，1.0=正常 |
| 销量来源 | `company.last_quarterly.sales_qty` | 已有字段，上季实际销售量 |
| 展示位置 | 经营指标网格第 6 项（库存之后） | 需求量是运营核心指标，与产能/库存平级 |
| 颜色 | 景气>1 → `text-accent-red`（涨红），景气<1 → `text-accent-green`（跌绿） | 与 A 股颜色约定一致（涨红跌绿） |
| 无数据 | 上季无结算时仅显示景气偏离 | 公司刚成立时 last_quarterly 为空 |
| 悬停提示 | 不展示 | 保持简洁 |

## Risks / Trade-offs

- **无上季结算时**：`last_quarterly` 为 null，退化仅显示景气偏离，不抛错
- **需求 vs 销量混淆**：指标名"需求量"实际显示的是上季销量（`sales_qty`），不是理论需求（`demand`）。因为销量是玩家可直接感知的数值，而理论需求包含了未满足的部分
