# 景气度随机游走改进：正态分布步长 + 非对称边界

## 问题

当前 `WalkProsperity()` 使用**均匀分布**步长 `(rand.Float64()*2 - 1) * MaxStep`，导致每个季度景气度变化的幅度完全均等，缺乏"大部分小波动、偶尔大波动"的真实经济周期感。

此外，当前各行业景气度边界是对称的（如 `[0.85, 1.15]`），但景气度围绕 1.0 做**乘法意义上的游走**（传导到售价是 `prosperity^elasticity`），对称边界意味着下行比例空间和上行比例空间不对称：

- 从 1.0 到 0.85 = **-15%**
- 从 1.0 到 1.15 = **+15%** ✓ 看起来对称

但比例上看：如果景气度从 1.0 跌到 0.85，需要涨 `1/0.85 - 1 = 17.6%` 才能回到原点。也就是说，**下行 -15% 需要上行 +17.6% 才能弥补**，对称边界实际造成下行更容易触及、上行更难弥补的不平衡。

## 目标

1. **正态分布步长**：用 `rand.NormFloat64()` 替换均匀分布，大部分季度微幅波动（±1σ 以内），少数季度大幅波动（±2σ~3σ），更贴近真实经济周期
2. **非对称边界**：`max = 1/min`，使上行和下行比例空间对称。如 `min=0.85 → max=1/0.85≈1.1765`，下行 15% 对应上行 17.6%，比例等价
3. **偏差归一化修正**：边界非对称后，偏离度 `deviation` 的计算需区分上方和下方，正确归一化到 [-1, +1]

## 范围

### 包含

| 范围 | 说明 |
|------|------|
| `prosperity.go` | 重写 `WalkProsperity()` 算法 |
| `industry.go` | 更新各行业参数：`ProsperityMaxStep` → `ProsperityStdDev`，`ProsperityMax` 为非对称值 |
| 边界 clamp | 仍然保留，防止极端随机值出界 |
| 回归与脉冲 | 保留向心回归和修正脉冲，逻辑适配非对称边界 |

### 不包含

- 前端显示改动
- `simulate_v2.py` 模拟器
- 景气度传导公式（PriceElasticity、需求算法等）
- 数据库 schema 变更（`IndustryProsperity` 表结构不变）
