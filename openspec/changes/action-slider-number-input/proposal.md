# 行动滑块增加数字输入框

## 问题

在经营行动弹窗中，所有操作（扩建/招募/裁员/出售资产/营销/注资/分红）都只通过 `<input type="range">` 滑块来调整数值。当数值范围很大时（如注资几百万、扩建费用过亿），滑块的分辨率严重不足：

- 滑块步进为 1，但拖动时每 1px 可能代表几万甚至几十万
- 用户难以精确输入想要的数值，只能反复拖动微调
- 分红场景也需要精确到分（0.01），滑块很难精确定位

## 目标

为行动数量输入提供一个**滑块 + 数字输入框**的组合控件，保留滑块做快速视觉调整的同时，支持键盘直接输入精确数值。

## 范围

### 包含

| 范围 | 说明 |
|------|------|
| `CompanyPage.tsx` 行动弹窗 | 替换当前纯滑块为双向同步的滑块+输入框组合 |
| 输入校验 | 失焦时 clamp 到 [0, maxAmount]，避免越界提交 |
| 分红精度 | 输入框同步保留 `step=0.01` + `toFixed(2)` 精度 |

### 不包含

- 公司创建环节的两个滑块（玩家投资金额、投资者股份）—— 数值范围有限，暂不需要
- IPO 弹窗的增发比例滑块 —— 范围 10%-50%，步进 1%，分辨率够用
- 可复用的通用 `SliderInput` 组件提取 —— 如果后续多处用到再重构
- 键盘快捷操作（如 Enter 提交、Esc 关闭）
